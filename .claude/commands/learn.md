# /learn - 再利用可能なパターンの抽出

現在のセッションを分析し、スキルとして保存する価値のあるパターンを抽出します。

## トリガー

セッション中に非自明な問題を解決した時点で `/learn` を実行してください。

## 抽出対象

以下を探します:

1. **エラー解決パターン**
   - どのようなエラーが発生したか？
   - 根本原因は何だったか？
   - 何で修正したか？
   - 同様のエラーに再利用できるか？

2. **デバッグ手法**
   - 非自明なデバッグ手順
   - 有効だったツールの組み合わせ
   - 診断パターン

3. **ワークアラウンド**
   - ライブラリの癖
   - APIの制限事項
   - バージョン固有の修正

4. **プロジェクト固有のパターン**
   - 発見したコードベースの規約
   - 決定したアーキテクチャ方針
   - 統合パターン

## 出力形式

`~/.claude/skills/learned/[pattern-name].md` にスキルファイルを作成します:

```markdown
# [パターンの説明的な名前]

**Extracted:** [日付]
**Context:** [適用場面の簡潔な説明]

## Problem
[解決する問題 - 具体的に記述]

## Solution
[パターン/手法/ワークアラウンド]

## Example
[該当する場合はコード例]

## When to Use
[トリガー条件 - このスキルを適用すべき状況]
```

## プロセス

1. セッションから抽出可能なパターンを確認する
2. 最も価値があり再利用性の高い知見を特定する
3. スキルファイルの草案を作成する
4. 保存前にユーザーに確認を求める
5. `~/.claude/skills/learned/` に保存する

## 注意事項

- 些細な修正（タイポ、単純な構文エラー）は抽出しない
- 一回限りの問題（特定のAPI障害など）は抽出しない
- 将来のセッションで時間を節約できるパターンに注力する
- スキルは焦点を絞る - 1つのスキルに1つのパターン
