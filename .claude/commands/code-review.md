# コードレビュー

未コミットの変更に対する包括的なセキュリティおよび品質レビュー:

1. 変更ファイルの取得: git diff --name-only HEAD

2. 各変更ファイルに対して以下をチェック:

**セキュリティ問題 (CRITICAL):**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力バリデーションの欠如
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質 (HIGH):**
- 50行を超える関数
- 800行を超えるファイル
- 4階層を超えるネスト
- エラーハンドリングの欠如
- console.log の残存
- TODO/FIXME コメント
- パブリックAPIに対するJSDocの欠如

**ベストプラクティス (MEDIUM):**
- ミューテーションパターン (代わりにイミュータブルを使用)
- コード/コメント内の絵文字使用
- 新規コードに対するテストの欠如
- アクセシビリティ問題 (a11y)

3. 以下を含むレポートを生成:
   - 重要度: CRITICAL, HIGH, MEDIUM, LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 修正の提案

4. CRITICAL または HIGH の問題が見つかった場合はコミットをブロック

セキュリティ脆弱性のあるコードは絶対に承認しないこと!
