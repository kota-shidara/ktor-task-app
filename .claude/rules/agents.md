# エージェントオーケストレーション

## 利用可能なエージェント

`~/.claude/agents/` に配置:

| エージェント | 目的 | 使用タイミング |
|-------|---------|-------------|
| planner | 実装計画の策定 | 複雑な機能開発、リファクタリング |
| architect | システム設計 | アーキテクチャに関する意思決定 |
| tdd-guide | テスト駆動開発 | 新機能開発、バグ修正 |
| code-reviewer | コードレビュー | コード記述後 |
| security-reviewer | セキュリティ分析 | コミット前 |
| build-error-resolver | ビルドエラーの修正 | ビルド失敗時 |
| e2e-runner | E2Eテスト | 重要なユーザーフロー |
| refactor-cleaner | 不要コードの整理 | コードメンテナンス |
| doc-updater | ドキュメント管理 | ドキュメント更新時 |

## 即時エージェント使用

ユーザーの指示を待たずに使用する場合:
1. 複雑な機能リクエスト - **planner** エージェントを使用
2. コードの記述・修正直後 - **code-reviewer** エージェントを使用
3. バグ修正または新機能 - **tdd-guide** エージェントを使用
4. アーキテクチャの意思決定 - **architect** エージェントを使用

## 並列タスク実行

独立した操作には必ず並列 Task 実行を使用すること:

```markdown
# 良い例: 並列実行
3つのエージェントを並列起動:
1. Agent 1: auth.ts のセキュリティ分析
2. Agent 2: キャッシュシステムのパフォーマンスレビュー
3. Agent 3: utils.ts の型チェック

# 悪い例: 不要な逐次実行
まず agent 1、次に agent 2、最後に agent 3
```

## 多角的分析

複雑な問題に対しては、役割を分担したサブエージェントを使用する:
- 事実確認レビュアー
- シニアエンジニア
- セキュリティ専門家
- 整合性レビュアー
- 冗長性チェッカー
