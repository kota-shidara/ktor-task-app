# テスト要件

## 最低テストカバレッジ: 80%

テストの種類（全て必須）:
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **統合テスト** - APIエンドポイント、データベース操作
3. **E2Eテスト** - 重要なユーザーフロー（Playwright）

## テスト駆動開発

必須ワークフロー:
1. まずテストを書く（RED）
2. テストを実行する - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行する - 成功するはず
5. リファクタリングする（IMPROVE）
6. カバレッジを確認する（80%以上）

## テスト失敗時のトラブルシューティング

1. **tdd-guide** エージェントを使用する
2. テストの分離性を確認する
3. モックが正しいか検証する
4. テストではなく実装を修正する（テストが間違っている場合を除く）

## エージェントサポート

- **tdd-guide** - 新機能開発時に積極的に使用、テストファースト手法を強制
- **e2e-runner** - Playwright による E2E テストの専門エージェント
